# RAG A/B 평가 리포트 — 2026-05-27

## 평가 목적
RAG 파이프라인 개선(쿼리 확장 강화 + FTS OR 검색) 전후 성능 측정.  
골든셋: LLM 판단 기반 25개 질의 (`golden_ab_test.jsonl`) — 키워드·자연어·로드맵·전공 직무 혼합.

## 파이프라인 정의

| Pipeline | 설명 |

## 주요 개선 원인

1. **Issue 1 Fix — 쿼리 확장 확대**: `classify_query_and_expand`의 3-case 하드코딩 → `expand_query_single_string` 호출. 자격증 동의어(정처기→정보처리기사), 직무/전공/구어 60+ 패턴 활성화.

2. **Issue 2 Fix — FTS OR 검색**: 확장된 긴 쿼리에 `plainto_tsquery` AND 사용 시 매칭 0건 문제 → `to_tsquery('simple', 'term1 | term2 | ...')` OR 검색으로 Sparse 채널 재활성화.

3. **MRR +35.7%**: 확장 덕분에 정답 자격증이 결과 상위에 더 빨리 등장함. 사용자에게 첫 화면에서 정답이 보임.

4. **레이턴시 -61%**: Sparse 채널 쿼리가 OR로 단순화되면서 DB 실행 계획 최적화 + OpenAI 임베딩 캐시 히트율 향상.

## 결론 및 환경변수 결정

**current 파이프라인이 모든 지표에서 우세하며 레이턴시도 더 빠름 → 현행 설정 유지.**

변경이 필요한 환경변수: **없음** (코드 레벨 개선이므로 `.env` 수정 불필요).

옵션으로 확인할 수 있는 추가 튜닝 변수:

```env
# Hybrid RAG 가중치 (현행 기본값으로 충분)
RAG_CURRENT_W_D=1.3   # Dense 가중치 (자연어 질의에 적합)
RAG_CURRENT_W_S=0.7   # Sparse 가중치

# BM25 인덱스 설정 (현행 권장값)
RAG_BM25_USE_KOREAN_NGRAM=true
RAG_BM25_NAME_BOOST=true
RAG_BM25_K1=1.5
RAG_BM25_B=0.75
```
